# Data

Parent rules: `docs/RULES.md` · Architecture: `docs/ARCHITECTURE.md`

## Purpose

All persistent data. JSON files on disk are the source of truth. config.json holds runtime configuration. state.json holds system state (created at runtime, gitignored). Output files are generated at the end of a run.

## Rules

- Files over databases — all state is JSON on disk
- config.json is the only manually edited file
- state.json is managed by core/hub.py — do not edit manually
- Output files are generated by export functions

## Files

| File | Purpose | Depends On |
|------|---------|------------|
| `config.json` | Runtime configuration: models, presets (per-mode tier→model), agent_overrides, turn_limits (per-mode), organization, model_mode, max_concurrent_api_calls | — |
| `state.json` | System state (gitignored, created at runtime) | Managed by `core/hub.py` |
